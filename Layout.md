# Specifics
For all areas: if they are irrelavant they should be closed. Practically: agent and roadmap area should be visible during iterations. When devteam is in plan stage it should be a flow through the feedback area (which has more space)

## Agent area
An overview of each running agent and the task they are working on

## Roadmap area
Tasks in scope marked as [X] for complete or [ ] for open, decided by orchestrator/planner/architect priority. When all are done: new list

## Feedback area
Information about open questions, and general devteam feedback (status, next steps, commands), is it possible to have its own scrolling?. And when planning this is where the 'chat' with the user should take place. Should feel as an actual agent chat.

## Input area
Here is where the user types
